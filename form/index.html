Squi\Form

Laravel has a fantastic set of form element helpers (and as of 3.1, the really cool macro feature), but Squi takes it a step further to allow you to generate an entire laid-out form in just a few lines.

Simple form

Let's start with a form we're all familiar with: a user registration form. Here are some basics demonstrating text, password and dropdown fields.

PHP

$fields = array(
  'name',
  'email',
  'password:password',
  'password2:password' => 'Confirm Password',
  'user_source:select' => array(
    'label' => 'How did you hear about us?',
    'options' => array(
      'friend' => 'From a friend',
      'google' => 'Google search',
      'other' => 'Other'
    ),
  ),
);

$form = Squi\Form::make($fields)->with('action', 'users/new');

Applying attributes to fields and labels

		
So often when you're displaying a table, you aren't just spitting out the contents of a database table. You probably want to format that data, maybe concatenate some fields, add conditional CSS classes, etc. This example shows how this can be done with Squi.

      
Many parameters allow you to provide closures, which you can use to return parameters based on their context. The value parameter of a column can be specified as a closure accepting a reference to the $row object belonging to the current table row. The same goes for most of the _attr parameters, opening up some neat possibilities for customization.

		

    

      

        

          
          
        

        

          
          
        

        

          
          
        

        

          
          
        

        

          
          
        

        

          
          
        

        

          
        

      

    

  

	





	

		
PHP

		
$fields = array(
  'name',
  'email' => array(
    'required',
    'value' => 'example@example.com',
  ),
  'password:password',
  'password2:password' => 'Confirm Password',
  'user_source' => array(
    'type' => 'select',
    'label' => 'How did you hear about us?',
    'label_attr' => array('style' => 'font-weight:bold;'),
    'options' => array(
      'friend' => 'From a friend',
      'google' => 'Google search',
      'other' => 'Other'
    ),
  ),
  'buttons' => array(
    'submit' => array(
      'text' => 'Sign Up',
      'attr' => array('class' => 'btn btn-primary'),
    ),
  ),
);

$form = Squi\Form::make($fields)
  ->with('action', 'users/new')
  ->with('method', 'get');

Layouts

      
By changing the layout property, you can have Squi render the form with your desired layout, currently with support for basic, table and bootstrap. Those provide a solid base for extension by CSS. While the bootstrap layout is named after the Twitter Bootstrap, it's actually just for twitter bootstrap "Horizontal" forms; The basic layout in conjunction with the proper classes provides excellent support for the other bootstrap form styles.

Integration with Models

   
If you find yourself displaying a particular model in a table multiple times, you might consider storing the column configuration inside the model declaration. Squi::of($class, $method) allows to pass a model class name, with Squi automatically querying a static method of that class for the column configuration array. The default method (if one is not specified) is table_columns($table). Notice that the method is passed a reference to the $table it is generating, allowing you to modify the column configuration based on table configuration such as layout or class.



   

      
Model Declaration

      
public static function form_fields($form)
{
   return array(
      'name' => 'Client Name',
      'birth_date' => array(
         'heading' => 'Birthdate',
         'value' => function($row)
         {
            return date('d-M Y', strtotime($row->birth_date));
         },
      ),
   );
}


   

   

      
Controller Code

      
$clients = Client::all();

$table = Squi\Table::of('Client')->with('rows', $clients);


      
You could also add additional methods to your model to configure alternate column configurations, like maybe a summary table that only includes a few columns. You would then call upon those methods like so:

      
$table = Squi\Table::of('Client', 'summary_table')
   ->with('rows', $clients);


   








   
The kitchen sink

   
There are a bunch of different ways to declare the configuration of Squi Tables, so here's a jumbled mess of most of them. A more organized listing of available options follows.



   

      
Using with( ) options

      
$table = Squi\Form::make()
  ->with('layout', 'table')
  ->with('form_attr', array('class' => 'table table-striped'))
  ->with('field_attr', function($field)
  {
    return array('data-uri' => 'clients/'.$row->id);
  })
  ->with('heading_attr', array('style' => 'color:red;'))
  ->with('norecords_message', 'Sorry, no rows to display')
  ->with('columns', array(
    'name' => array(
      'heading' => 'Client Name',
      'heading_attr' => array('width' => 300),
      'value' => function($row)
      {
        return $row->firstname.' '.$row->lastname;
      },
      'cell_attr' => array('class' => 'client-cell'),
    ),
  ));


   

   

      
Shorthand

      
Squi\Form::make($fields, $instance, 'bootstrap');

Squi\Form::basic($fields, $instance);

Squi\Form::table($fields, $instance);

Squi\Form::bootstrap($fields, $instance); 

Squi\Form::of('Client', $instance, 'table');

Squi\Form::of($client_instance);


      
All of these shorthand methods return the Form instance to allow method chaining.
